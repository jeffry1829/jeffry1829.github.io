2018/8/1 //關於LCA求MST(Minimal Spanning Tree)(最小生成樹)兩點唯一路徑權重最大值
和上一篇一樣，我還是在想TIOJ1163，我看過解答，我始終想不透為何麼LCA可以快速解這問題，
結果只是我想多了，其實想法異常粗暴。(EDIT:其實也要用倍增法，詳見下篇文章)

這題目大部分會被寫在求次小生成樹的解法上，因為次小生成樹的解法是:
先求MST(kruskal算法)，並一一嘗試每一個剩下沒連接到的邊(u,v,w(weight))， 必定形成環，檢查(u,v)間另外一條路徑上的最大權重，刪掉這edge， 接上我們正在嘗試的edge(u,v,w)，會得到新的樹權重和，最後輸出權重最小的結果就好。

回到正題，這個問題的解法是:
->let LCA=lca(u,v)
->define walk(u,v):
->->assume depth(u)>depth(v) and v is an ancestor of u
->->因為在MST,(u,v)間只有一條路徑，所以我們可以一步一步往u上面的祖先走，並記錄最長的邊(可以用倍增法 ，詳見下篇文章)
->->得到(u,v)間的最大權重
->end walk
->(因為walk傳入的(u,v)必須是祖先-子孫關係，所以我們必須先求LCA)
->現在我們就可以得到答案是max(walk(u,LCA),walk(v,LCA))
->end

2018/8/21 //Codeforce 510C

Key: Topological sort(拓樸排序)
大約意思是給定一張DAG(Directed Acyclic Graph有向無環圖)，把所有點展開成連的邊只往下指 ，輸出一種點的順序滿足點是從最上"層"到最下"層"。
至於判斷是否是DAG的方法其中一種是利用dfs，紀錄每一點的離開時間，如果 dfs到其中一點u時，發現u連到一個點v，但v已進入卻"未離開"，即代表有環存在。 拓樸排序也可以簡單的由輸出離開時間從最晚到最早得到。

//Codeforce 543B
首先要意識到這問題等價於求出滿足條件需要最少的edge數
接著可以分成兩種情況，1.兩條路徑不相交2.兩條路徑有重疊(邊重疊)
bfs求出全點對最短路(因為邊權為一，不需要特別的最短路演算法)，
對於第一種情況的路徑邊數為$$dis(s1,t1)+dis(s2,t2)$$(dis:(x,y),x,y的最短路徑長度)，
至於對第二種情況，遍歷所有s1t1,s2t2可能的重複路徑(i,j)，取其最小(和狀況一一起比大小)，最終總邊數扣掉最少需要邊數即為解答。

2018/10/10 //加入Webgl Live2D
調適了一整天，終於完成了。以下列出步驟:
1.下載Live2D SDK 外部連結
2.將sdk中的lib,framework和sample/sampleApp1/src,assets都複製到網頁目錄裡
3.因為調配過程中會用到cross-site request，所以隨便裝個http server來serve你的頁面，我使用Windows miniweb，非常輕巧。(網址:localhost:8000)
4.在你自己寫的html裡加上
<body onload="sampleApp1()">
<button id="btnChange" class="active" style="position: fixed; float:right; z-index: 101; right:3px; bottom: 480px;">Change Model</button>
<canvas id="glcanvas" width="340px" height="470px" style="position: fixed; float:right; z-index: 100; right:0px; bottom: 0px;"></canvas>
(注意不要用css指定width/height，因為sdk是去抓html tag的數值，不是css的)
5.head tag裡加入外部連結，(path需要自己調整一下， live2d/..... 是我的資料夾配置)
6.基本上已經有預設的4種顯示了。

接下來講解如何加入新顯示。
1.首先介紹一個分享model的網站 外部連結
2.本次教學使用外部連結作為範例。
3.下載後整個資料夾丟入assets/live2d，我們要使用的是ev2中的model。
4.開啟src/LAppDefine.js，加入MODEL_HOUKAGOU : "/live2d/assets/live2d/houkago2/ev2/moc/ev_02/ev02.model.json",
5.開啟src/LAppLive2DManager.js，找到LAppLive2DManager.prototype.changeModel，裡面有一個switch(no)，大概意思就是每按一次按鈕no+=1然後處理不同的case，可以稍微讀讀。
6.把上面var no = parseInt(this.count % 4);改成%5，因為我們要多加一個進去。
7.把case 3改成這樣外部連結，因為原本的case3有兩個model，所以下一個model需要release兩次，為了方便，我們直接把case3改成新model。(我在背景下了一點功夫，可以讀一讀。path一樣需要調整一下)
8.但是會發現人物太小了，和預期的效果不同，此時我們需要到assets/live2d/houkagou2/ev2/moc/ev_02/ev02.model.json改一下顯示大小。
9.加入
"layout": { "center_x":0, "width":2.2 }
width好像並沒有一個嚴格的比例對應到我們canvas的大小，所以之後調新model的時候可以自己試試看哪樣合適。
10.大功告成。
